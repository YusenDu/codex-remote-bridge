# 命令活动卡片与实时状态中文化设计

## 目标

在不改变 Codex Desktop 原始消息、命令执行和展开逻辑的前提下：

1. 将连续命令分组改成接近 LiveAgent `Tool Activity` 的紧凑卡片结构。
2. 将实时活动区域的固定状态和常见英文 reasoning 摘要中文化。
3. 保留无法可靠翻译的原文，避免误译路径、命令、模型名和代码。

## 命令活动卡片

连续命令仍由现有分组算法生成，不改变消息排序和命令输出数据。

折叠标题行包含：

- 命令图标和“命令活动”标题。
- `N 条`数量标签。
- `Shell N` 类型摘要。
- 右侧总状态：成功、运行中、失败或已中断。
- 展开箭头。

展开后每条命令使用独立紧凑行，包含 Shell 图标、命令摘要、状态点和状态文字。点击命令行继续展开原有输出。运行中与失败分组自动保持展开，全部成功后允许自动收起；用户手动切换后尊重用户选择。

样式使用当前页面的深色变量，圆角不超过 8px，不引入 LiveAgent 的浅色背景或 React 依赖。

## 实时状态中文化

固定活动状态使用确定性映射，例如：

- `Thinking` -> `正在思考`
- `Planning` -> `正在规划`
- `Running command` -> `正在运行命令`
- `Writing response` -> `正在整理回复`

Reasoning 摘要只在展示层处理。翻译器按行工作：

1. 已包含中文的行保持不变。
2. 保护反引号代码片段、路径、URL、模型名和大写标识符。
3. 优先匹配完整常见短语，再翻译句首动作词和常见技术名词。
4. 没有可靠匹配时返回原文。
5. 不写回 store，不修改线程历史，不向外部翻译服务发送内容。

首版覆盖规划、检查、读取、搜索、运行、构建、测试、验证、更新、等待和整理回复等常见状态。

## 组件边界

- `src/utils/liveActivityLocalization.ts`：纯函数中文化，独立单元测试。
- `ThreadConversation.vue`：调用本地化函数并渲染新的命令活动结构。
- 现有命令分组、输出展开、消息分页和实时状态 store 保持不变。

## 验证

- 单元测试覆盖固定状态、常见摘要、中文原文、代码与路径保护、未知短语回退。
- 组件契约测试覆盖命令活动标题、数量、类型摘要、状态和本地化调用。
- 运行完整 Web 测试与生产构建。
- 在公网任务页面验证折叠、展开、单条输出、运行中、成功和失败样式；检查桌面与移动宽度、控制台和文本溢出。

## 非目标

- 不调用翻译 API。
- 不翻译已持久化的历史正文。
- 不改变 Codex Desktop 的 reasoning 语言。
- 不迁入 LiveAgent 的 React 组件或 Agent 运行时。
