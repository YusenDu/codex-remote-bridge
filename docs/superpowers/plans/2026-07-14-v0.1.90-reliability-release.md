# Codex Remote Bridge v0.1.90 Reliability Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver Web and Windows Agent v0.1.90 with targeted thread synchronization, reliable remote RPC, hidden internal screenshots, stable branding, update checking, administrator installation, tests, packages, and GitHub publication.

**Architecture:** Keep Web synchronization event-driven and store dirty state per thread. Reduce large RPC payloads inside the Codex renderer adapter before they cross the desktop relay. Keep desktop updates explicit by checking GitHub Releases and opening the matching release, while NSIS handles elevated system-wide installation.

**Tech Stack:** Vue 3, TypeScript, Vitest, Node.js/Express/WebSocket, Rust, Tauri 2, NSIS, GitHub Releases.
